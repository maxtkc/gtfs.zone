🚀 Getting Started

Quick Start Guide

  1. Create a new feed: Click "New" to start with sample GTFS data
  2. Upload existing data: Click "Upload" or drag & drop a GTFS ZIP file
  3. Explore your data: Use the "Objects" tab to navigate agencies, routes, trips, and stops
  4. Edit files: Click any file in the "Files" tab to edit with syntax highlighting
  5. Visualize on map: See your transit data plotted automatically on the map
  6. Export: Click "Export" to download your modified GTFS feed

✨ Features

🗺️ Interactive Map

Visualize stops, routes, and shapes on an interactive map. Search for specific stops or routes using the map search.

📝 Smart Editor

Edit GTFS files with CSV syntax highlighting. Switch between text and table views for easier editing.

🔍 Data Explorer

Navigate your GTFS data hierarchically: Home → Routes → Trips → Stop Times. Search and filter objects.

✅ Validation

Automatic GTFS validation with detailed error reports, warnings, and feed statistics.

📄 GTFS File Guide

Required Files

• agency.txt - Transit agencies
• routes.txt - Transit routes
• trips.txt - Individual trips
• stops.txt - Stop locations
• stop_times.txt - Trip schedules
• calendar.txt - Service dates

Optional Files

• shapes.txt - Route geometry
• calendar_dates.txt - Service exceptions
• fare_attributes.txt - Fare information
• fare_rules.txt - Fare rules
• frequencies.txt - Headway-based service
• transfers.txt - Transfer rules

💡 Tips & Tricks

  • • Large files: Use the table view for easier editing of large CSV files
  • • Validation: Check the Info tab for validation results and feed statistics
  • • Map navigation: Click on stops or routes in the Objects tab to highlight them on the map
  • • Search: Use the search bar on the map or in the Objects tab to find specific data
  • • Export format: Exported files maintain the original structure and can be used with any GTFS-compatible software
